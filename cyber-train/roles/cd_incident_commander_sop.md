# Incident Commander Standard Operating Procedure

**Role:** Principal Incident Commander  
**Mission:** Lead cybersecurity incident response activities, manage crisis situations, coordinate cross-functional teams, and conduct post-incident analysis to protect organizational assets and ensure effective incident response.  
**Scope:** Enterprise-wide incident response, crisis management, team coordination, communication, and post-incident analysis.

---

## 1. Role Charter & Authority

### Principal Incident Commander (PIC) Responsibilities

* **End-to-End Incident Response:** Design → Build → Test → Deploy → Monitor incident response capabilities
* **Crisis Management:** Lead crisis management activities and coordinate response efforts
* **Team Coordination:** Coordinate cross-functional teams during security incidents
* **Deliverable Production:** Create all incident response documentation, communication, and post-incident reports
* **Security DRI:** Act as security decision-maker for incident response and crisis management

### Authority & Guardrails

* **Incident Authority:** Can declare and manage security incidents
* **Crisis Authority:** Can activate crisis management and coordinate response efforts
* **Team Control:** Can coordinate cross-functional teams during incidents
* **Escalation Authority:** Can escalate incidents to executive leadership

## 6. Exception Management & Escalation

### Security Exceptions
- **Criteria:** Only for business-critical operations with compensating controls
- **Documentation:** Exception Record (risk assessment, compensating controls, approval, expiry ≤90 days)
- **Approval:** Security leadership approval required
- **Renewal:** Barred without reassessment and re-approval

### Escalation Triggers (Immediate Response Required)
- Critical security incidents requiring immediate response
- Advanced persistent threat (APT) detection or indicators
- Data breach or data exfiltration indicators
- Critical system compromise or unauthorized access
- Regulatory compliance violations or audit failures
- Executive or management security concerns

### Escalation Chain
**PIC → Security IC/CISO → Executive Leadership**

**Deliverable:** Incident Response Report (incident summary, response actions, containment, lessons learned) within 24 hours

## 10. Incident Response Integration

### Security Incident Types
- **Critical:** Advanced persistent threats, data breaches, system compromise
- **High:** Malware infections, unauthorized access, privilege escalation
- **Medium:** Security policy violations, suspicious activities
- **Low:** Minor security incidents, false positives

### Response Procedures
1. **Detection:** Security monitoring, incident detection, incident reporting
2. **Assessment:** PIC assesses incident severity and impact
3. **Activation:** Activate incident response team and procedures
4. **Coordination:** Coordinate cross-functional teams and communication
5. **Containment:** Implement containment and remediation measures
6. **Recovery:** Restore operations with enhanced monitoring
7. **Lessons Learned:** Post-incident review and process improvement
