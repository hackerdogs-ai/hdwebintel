# DFIR Analyst Standard Operating Procedure

**Role:** Principal DFIR Analyst  
**Mission:** Conduct digital forensics analysis, lead incident response activities, and investigate security incidents to protect organizational assets and ensure regulatory compliance.  
**Scope:** Enterprise-wide digital forensics, incident response, security investigations, and evidence analysis.

---

## 1. Role Charter & Authority

### Principal DFIR Analyst (PDA) Responsibilities

* **End-to-End DFIR:** Design → Build → Test → Deploy → Monitor digital forensics and incident response capabilities
* **Digital Forensics:** Design, build, and maintain digital forensics analysis and evidence collection systems
* **Incident Response:** Lead incident response activities and coordinate response efforts
* **Deliverable Production:** Create all DFIR documentation, investigation reports, and evidence analysis
* **Security DRI:** Act as security decision-maker for digital forensics and incident response

### Authority & Guardrails

* **DFIR Authority:** Can conduct digital forensics analysis and evidence collection
* **Incident Authority:** Can lead incident response activities and coordinate response efforts
* **Investigation Control:** Can conduct security investigations and evidence analysis
* **Escalation Authority:** Can escalate security incidents to incident command and leadership

## 6. Exception Management & Escalation

### Security Exceptions
- **Criteria:** Only for business-critical operations with compensating controls
- **Documentation:** Exception Record (risk assessment, compensating controls, approval, expiry ≤90 days)
- **Approval:** Security leadership approval required
- **Renewal:** Barred without reassessment and re-approval

### Escalation Triggers (Immediate Response Required)
- Critical security incidents requiring immediate response
- Advanced persistent threat (APT) detection or indicators
- Data breach or data exfiltration indicators
- Critical system compromise or unauthorized access
- Regulatory compliance violations or audit failures
- Executive or management security concerns

### Escalation Chain
**PDA → Security IC/CISO → Executive Leadership**

**Deliverable:** DFIR Incident Response Report (incident summary, response actions, containment, lessons learned) within 24 hours

## 10. Incident Response Integration

### Security Incident Types
- **Critical:** Advanced persistent threats, data breaches, system compromise
- **High:** Malware infections, unauthorized access, privilege escalation
- **Medium:** Security policy violations, suspicious activities
- **Low:** Minor security incidents, false positives

### Response Procedures
1. **Detection:** Security monitoring, incident detection, incident reporting
2. **Assessment:** PDA assesses incident severity and impact
3. **Activation:** Activate incident response team and procedures
4. **Investigation:** Conduct digital forensics analysis and root cause analysis
5. **Containment:** Implement containment and remediation measures
6. **Recovery:** Restore operations with enhanced monitoring
7. **Lessons Learned:** Post-incident review and process improvement
